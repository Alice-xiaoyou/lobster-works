系统架构：四个模块

模块1：市场侦察兵。 每天扫HN、工具定价、行业新闻。用HN Algolia API——免费、不需Token。中文平台有反爬，英文社区可以稳定抓。成本：$0。

模块2：战略大脑。 AI分析信号交叉验证。置信度门槛0.8。4天捕捉12个信号，6个被多源验证。Vibe Coding中文真空就是被4个独立信号同时打中才确认为主攻方向。成本：$0.03/次。

模块3：内容工厂。 AI生成四类：知乎长文、小红书短文案、即刻短文、产品描述。每篇有质量gate——必须有可追溯数据源。

最大的坑在这里：Day 1 AI狂产18篇，后来发现8篇的「实测数据」是编的——提示词要求「基于实测」，AI没有实测能力就脑补了。全废。AI内容系统的瓶颈不是产量，是判断力。

模块4：分发管线。 这也是坑——小红书内容填好了40小时，发布按钮是Vue自定义组件，自动化工具点不了。管线不是代码问题，是流程问题。

最小启动方案（10分钟，$0）

需要：一台电脑 + DeepSeek API Key + 文本编辑器。

不要：服务器、数据库、前端框架、付费SaaS。

三步：

  1. 创建项目文件夹，放三个文件——state.json、strategy.json、AGENTS.md
  2. 配置cron：每6h侦察、每8h生产、每24h复盘
  3. 写Prompt，三条核心指令：
  • 每个数据点必须可追溯到来源
  • 做不到实测就诚实标注
  • 同一角度不超2批内容

四个花钱买不到的教训

1. AI会编数据，而且编得很真。 不给它实测工具它不会说做不到——它会脑补。解法：每篇内容必须标注数据来源。

2. 产量≠发布。 71篇写 vs 1篇发。AI能写，不能发。

3. 同质化是无声杀手。 Day 1-2产出一堆角度重复的内容。定规则：每个新批次必须开新角度或新赛道。

4. 小红书自动化比想象难10倍。 Vue自定义组件抓不到。发布按钮目前还需要人点。
