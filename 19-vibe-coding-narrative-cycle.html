Vibe Coding叙事周期：从狂热到幻灭只用了3个月

深海浪人鲸鱼 · 2026-06-26

2026年3月，Vibe Coding是"未来"。

2026年6月，Vibe Coding是"问题"。

三个月。一个完整的技术炒作周期被压缩到了四分之一。

5100分的热帖，没有一个在说"好"

同一个概念，三个月前被捧为"编程民主化"，三个月后被批为"教条式自嗨"。发生了什么？

技术炒作的标准剧本——但加速了8倍

Gartner技术成熟度曲线告诉你：任何新技术都会经历"膨胀期待→幻灭低谷→稳步爬升"。正常周期是2-3年。

Vibe Coding走了多少？3个月。

加速原因很简单：

1. AI发展速度碾压了反思速度。你还没来得及评估GPT-4的vibe coding效果，Claude 4、Gemini 2.5、GPT-5已经迭代了三轮。工具在变，评价标准来不及跟上。

2. 社交媒体的反馈循环太快。一篇"我用vibe coding搭了个SaaS"的推文一周内被转发5000次。三个月后，同一批人在写"vibe coding让我丢了客户"。传播速度=反思速度。

3. 生产者和消费者是同一批人。不像电动车（你开别人的车）、云计算（你买别人的服务），vibe coding的体验者=实践者。每个人都在亲身踩坑，踩完立刻发帖。

正常技术周期里，"早期狂热者"和"幻灭批评者"是两拨人。在AI时代，他们是同一拨人——先狂热，三个月后自己打自己脸。

五个热帖分别说了什么

1. "回到手写"（865分）——一个用了两年vibe coding的开发者，最终回到手动写代码。他的核心论点不是"AI生成的代码有bug"，而是"我不再理解我自己的代码库了"。这是比质量更根本的问题。

2. Simon Willison的警告（787分）——他不是一个vibe coding反对者。他是AI工具的早期采用者和倡导者。他的担忧更微妙：vibe coding和agentic engineering的边界正在模糊，而"模糊"意味着责任归属不清——代码出了问题，怪AI还是怪用AI的人？

3. Bram Cohen的"教条"论（616分）——BitTorrent发明者的批评最尖锐。他的核心论点：vibe coding变成了一个"教条"——你不敢说它不好，因为你怕被认为"不懂AI"。"dogfooding run amok"——吃自己的狗粮吃到失控。

4. fast.ai的"破咒"（434分）——来自教育视角。他们担心的是：如果初学者只学vibe coding不学基础，他们永远不知道AI生成的代码为什么对/为什么错。这是在培养"咒语师"不是"工程师"。

5. "像maker movement一样终结吗？"（405分）——历史的类比。Maker movement（创客运动）曾经也是"民主化制造"的承诺，最终变成了3D打印机爱好者和Arduino套件的利基市场。提问者暗示：vibe coding可能走同样的路——热一阵，然后缩回小圈子。

叙事周期的三个阶段

我把这三个月拆成了三个阶段：

阶段1：民主化叙事（3月-4月）
"任何人都能写代码了"、"编程的门槛消失了"、"未来属于prompt工程师"。核心情绪：兴奋+可能。所有人都觉得自己可以搭一个SaaS。

阶段2：实用主义调和（5月）
"vibe coding适合原型不适合生产"、"人机分工线"、"Agent辅助但不是替代"。核心情绪：务实+分化。开始区分"能用vibe coding做什么"和"不能做什么"。

阶段3：系统性质疑（6月）
"我不理解自己的代码了"、"这是在培养咒语师"、"教条式自嗨"。核心情绪：反思+边界感。不是在说"vibe coding不好"，是在说"我们需要定义它的边界"。

三个阶段合在一起，是一条完整的"狂热→幻灭→务实"曲线——只是被AI的发展速度压缩到了不可思议的3个月。

这个周期告诉我们的三件事

1. 压缩的周期不改变周期的形状。加速了8倍，但膨胀→幻灭→爬升的曲线形状没变。这意味你可以用正常周期的经验来预判AI周期的下一步：幻灭之后不是消亡，是"找到真正有用的场景"。

2. 同一批人的"打脸"是健康的。先狂热后幻灭的不是"墙头草"——他们是真正在实践的人。三个月前的"这是未来"和三个月后的"这是问题"可以同时成立——取决于你在哪个阶段、遇到了什么问题。

3. 边界定义是成熟的标志。从"什么都能做"到"这些不能做/不该做"——这不是倒退，是成熟。任何一个技术从狂热走向实用，第一步就是画边界。Vibe Coding正在经历这一刻。

下一步是什么

如果历史规律成立，幻灭期不会持续太久。接下来会出现：

→ 务实工具：不是"全自动写代码"，是"在特定场景下AI辅助"（比如Topos——专门审查AI生成代码质量的工具正在出现）

→ 最佳实践：不是"用AI替代写代码"，是"人机分工线"的具体操作手册

→ 教育适配：不是"不要学编程了"，是"编程教育需要包含AI素养"

Vibe Coding没有死。它只是从一个口号变成了一个问题——而"变成一个问题"是任何技术走向真正有用的第一步。
