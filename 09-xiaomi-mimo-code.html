小米悄悄开源了一个AI编程Agent，拿了HN 556赞

深海浪人鲸鱼 | 速报 | 2026.06.21

今天HN上一个帖子冲到了556分：小米开源了 MiMo Code。

这不是手机评测。这是一个能处理200步以上复杂编程任务的AI Agent，而且声称在长周期任务上超越了Claude Code。

为什么这件事值得关注

第一，来源出人意料。 小米是一家手机/IoT公司，不是AI Lab。但他们推出的MiMo Code在编程Agent这个最卷的赛道拿了556个HN赞。这不是营销——HN社区对真实性的嗅觉极其敏锐。

第二，开源策略。 Claude Code是闭源的，Cursor是闭源的，Replit Agent也是闭源的。MiMo Code选择全部开源。在Google刚刚发布白皮书说「Agent = 10%模型 + 90%工具链」的同一周，小米把自己那90%的Harness代码公开了。

第三，中文AI工具在国际上被认真对待。 继DeepSeek后，这是一个新信号：中国公司在AI开发工具这个领域不是追随者。

与我们之前分析的对应

在之前的「人机分工线」框架里，我说过AI编程工具的核心差异不在模型层，在工程层。MiMo Code的定位恰好验证了这一点——它比拼的不是用了多强的模型，而是Agent的任务编排能力（如何处理200步以上的长链任务）。

在我们的Vibe Coding工具地图上：

  • Replit → 全民开发平台
  • Bolt → 专业设计驱动
  • Claude Code → 对话式编程
  • MiMo Code → 开源长周期Agent

这是第四种定位。

保持关注，暂不急判

和所有工具一样，我在实测之前不会给结论。但556个HN赞说明一件事：至少500多个用过的人觉得这东西是认真的。

等实测。
