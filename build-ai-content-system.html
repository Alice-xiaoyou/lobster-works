一、系统架构：四个模块怎么串起来的

模块1：市场侦察兵（Scout）
每天自动扫描Hacker News、工具定价页面、行业新闻。用的是HN Algolia API——免费、不需要Token、返回结构化JSON。中文平台有反爬保护不可行，但英文社区数据够用。真实成本：$0。

模块2：战略大脑（Strategy）
AI分析"哪些信号可以交叉验证""哪些是噪音"。系统内置信度门槛0.8。4天捕捉12个独立信号，6个被多数据源交叉验证。Vibe Coding中文教程真空是被4个信号同时打中才确认为主攻方向。成本：~$0.03/次分析。

模块3：内容工厂（Content Factory）
AI自动生成四类内容：知乎长文、小红书短文案、即刻短文、产品描述。每篇有质量gate——必须可追溯数据来源。

第一个大坑：Day 1狂产18篇，8篇"实测数据"是AI自己编的。提示词要求"基于实测"，AI没有实测能力就自动脑补。全部废掉重来。AI内容系统的瓶颈不是产量，是判断力。

模块4：分发管线（Distribution）
写完≠发布。平台注册、格式化、图片制作、发布按钮——每个都是真实卡点。小红书内容填好了40小时，发布按钮是Vue自定义组件自动化点不了。管线不是代码问题，是流程问题。

二、第一天就能跑起来的最小版本

你需要：一台电脑 + 一个AI API Key（DeepSeek最便宜）+ 文本编辑器 + 公开state.json。不需要服务器、数据库、前端框架、任何付费SaaS。

10分钟搭起来：

  1. 创建项目文件夹，放三个文件：state.json、strategy.json、AGENTS.md
  2. 配置cron job：每6h侦察兵扫描 + 每8h内容工厂 + 每24h战略复盘
  3. 给AI写好Prompt。核心三条：每个数据点必须可追溯到来源 / 做不到实测就诚实标注 / 同一角度不能产出超过2批内容
  4. 跑起来，公开日志。state.json公开是唯一无法被批量伪造的信任证明。

四、我踩过的坑

坑1：AI会编数据，而且编得很真

提示词写"基于真实测试"，AI不会说做不到，会直接脑补测试结果。对策：每篇必须有"来源标注"字段。

坑2：产量和发布是两个完全不同的能力

71篇产出 vs 1篇发布。AI能做的是"写"，做不到的是"发"。平台注册、格式适配、封面图、发布时机——每样都需要人。

坑3：内容同质化是无声杀手

Day 1-2产出了太多"分析蓝海"的内容，换汤不换药。定规则：每个新批次必须开辟新角度或新赛道。

坑4：小红书自动化发布比想象中难10倍

网页版创作后台Vue框架，发布按钮是自定义组件，自动化工具抓不到。填好了标题正文封面——就卡在"点击发布"。
